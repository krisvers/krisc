syntax:
almost all instructions use this style
[instruction] [destination], [value]
an example is mov
mov [dest], [value]
however, some such as set and clr appear as different instructions: ste, stc, sto, stg, stl, and, cle, clc, clo, clg, cll

instructions:
there are 11 macro instructions; mov, add, sub, cmp, jmp, mul, div, set, push, pop, and lod
in each of these instructions there are micro instructions, they're basically smaller variations or change up the destination/value use.

mov:
    mov [register], [immediate]
    mov [address], [register]

add:
    add [register], [value]
    add [address], [register]

sub:
    sub [register], [value]
    sub [address], [register]

cmp:
    cmp [register], [value]
    cmp [address], [register]

jmp:
    jmp [address]
    je [address]
    jg [address]
    jl [address]
    jge [address]
    jle [address]
    jc [address]
    jo [address]
    jne [address]
    jnc [address]
    jno [address]

mul: // product ends up in the e register
    mul [register], [value]
    mul [address], [register]

div: // quotient ends up in the f register
    div [register], [value]
    div [address], register

set:
    ste
    stc
    sto
    stg
    stl

    cle
    clc
    clo
    clg
    cll

push/pop:
    unfunctional at the moment

lod: // loads one byte at address into register
    lod [register], [address]

translation to binary:

macro instructions:

mov: 0b0000
add: 0b0001
sub: 0b0010
cmp: 0b0011
jmp: 0b0100
mul: 0b0101
div: 0b0110
set/clr: 0b0111
push/pop: 0b1000
lod: 0b1001

micro instructions:

if destination is memory add 0b1000
example: to move the value in d to an address: mov [address], d -> 0b00001011 ...
example: to subtract b from an address: add [address], b -> 0b00011010 ...

if destination isn't memory pad with a 0
example: moving value to f: mov f, [value] -> 0b00000101 ...

registers in micro instructions:

a: 0b000
b: 0b001
c: 0b010
d: 0b011
e: 0b100
f: 0b101
sp: 0b110
bp: 0b111

values:

16 bit value following micro opcode
